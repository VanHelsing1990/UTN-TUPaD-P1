MARCO TEÓRICO - SISTEMA DE GESTIÓN DE DATOS DE PAÍSES
=====================================================

1. ESTRUCTURAS DE DATOS EN PYTHON
----------------------------------

1.1 LISTAS
- Almacenan colecciones ordenadas de elementos
- Son mutables: se pueden modificar, agregar y eliminar elementos
- En el proyecto: lista principal que guarda todos los países
- Ejemplo: paises = [diccionario1, diccionario2, ...]

1.2 DICCIONARIOS
- Estructuras clave-valor para modelar entidades
- Cada país es un diccionario con 4 atributos
- Claves: nombre, poblacion, superficie, continente
- Acceso semántico: claves descriptivas en lugar de índices

2. PROGRAMACIÓN MODULAR
-----------------------

2.1 FUNCIONES ESPECIALIZADAS
- Cada función tiene una responsabilidad única
- Validación: validar_entero(), validar_texto()
- Archivos: cargar_paises(), guardar_paises()
- Lógica: agregar_pais(), buscar_pais(), actualizar_pais()

2.2 VENTAJAS DE LA MODULARIZACIÓN
- Mantenibilidad: cambios localizados
- Reusabilidad: funciones usadas múltiples veces
- Legibilidad: código auto-documentado
- Testabilidad: verificación independiente

3. ALGORITMOS DE ORDENAMIENTO
-----------------------------

3.1 ALGORITMO BURBUJA
- Ordenamiento por comparaciones sucesivas
- Compara elementos adyacentes y los intercambia
- Complejidad: O(n²) en el peor caso
- Estabilidad: mantiene orden de elementos iguales

3.2 CRITERIOS DE ORDENAMIENTO
- Nombre (A-Z y Z-A)
- Población (ascendente y descendente)
- Superficie (ascendente y descendente)
- Total: 6 criterios diferentes

4. GESTIÓN DE ARCHIVOS CSV
--------------------------

4.1 FORMATO CSV
- Valores separados por comas
- Legible 
- Estructura: nombre,poblacion,superficie,continente

4.2 MÓDULO CSV DE PYTHON
- Lectura/escritura segura de archivos
- Manejo automático de delimitadores
- Codificación UTF-8 para caracteres especiales

5. VALIDACIÓN Y MANEJO DE ERRORES
----------------------------------

5.1 TIPOS DE VALIDACIÓN
- Campos obligatorios vs opcionales
- Números enteros positivos
- Texto no vacío
- Unicidad de registros

5.2 ESTRATEGIAS
- Validación proactiva: antes del procesamiento
- Feedback inmediato al usuario
- Mensajes claros de error y éxito
